To perform the essential work of assigning topics to the content published there, the hub must acquire an understanding of the structure of the content and the vocabularies forming the topics. Collectively, this information is known as the content model, and whole models can be installed from files or other hubs. Alternatively, one can build up a model (or customize an imported one) using the links at the left to access the tools used to manage each model component.
