Define a content destination

Each destination represents an 'address' where content packages or notifications can be sent.

The first two selections specify how content will be delivered - only choose them if you can support the protocol: unless you run a repository, you are unlikely to have support for SWORD, but you almost certainly have an email client.

Choose a description that distinguishes one destination from another. The 'channelUrl' is an address: use an HTTP URL for SWORD, or a 'mailto:' URL for email. If authenticaton is required, enter the credentials also.
