Resource Maps

Most content consists of several related files - perhaps a metadata file and a few versions of the main document. To assist the hub in finding what it needs for topic extraction and other purposes, you must provide a map of all these resources of interest. Every published collection needs to be assigned a resource map. Note that the maps do not need to be exhuastive: just map the components needed, and expand later if desired.
